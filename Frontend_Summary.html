Gaffer's Guide: Frontend Architecture Study Guide

The frontend of Gaffer's Guide is a high-performance, completely decoupled Next.js application sitting inside the frontend_final/ package. It handles both the cinematic marketing website and the native-feeling desktop tactical workspace.

The Tech Stack

  • Core Framework: Next.js 16 (App Router) & React 19
  • Language: TypeScript for strict type-safety across components and API responses.
  • Styling: Tailwind CSS v4. We implemented a custom, strict "Dark Room" design system (bg-pitch, text chalk) combined with dynamic emerald-400 neon accents.
  • Animation Engine: A powerful trifecta:
    • Framer Motion: For buttery UI transitions and element enter/exit animations.
    • GSAP (GreenSock): For complex scroll-based timelines (scrollytelling).
    • Three.js (@react-three/fiber, @react-three/drei): For the physics and canvas background effects.

1. Project Shell & Infrastructure (frontend_final/)

The frontend operates as a strictly decoupled React project. Inside frontend_final/, you will find a dedicated package.json, Next.js configs, and a `.env.example` that specifically requests NEXT_PUBLIC_API_BASE=http://127.0.0.1:8000. This architecture ensures the frontend can be deployed to Vercel/Netlify completely independent of the FastAPI python backend.

2. Backend Integrations (frontend_final/lib/)

The UI is wired to real data constructs instead of using UI mocks:

  • lib/apiBase.ts: Resolves HTTP and WebSockets (wss://) targets dynamically from environment files.
  • lib/api/jobs.ts: A typed wrapper handling createJob(file) via FormData upload endpoints, and fetching artifacts via getTracking().

3. The Cinematic Marketing Website (frontend_final/app/page.tsx)

Located in /frontend_final/components/, this side uses extreme polish and data-driven aesthetics to sell the software.

  • OpeningSequence.tsx & TextReveal.tsx: Handle staggered cinematic text loads.
  • TransformationStages.tsx & ScrollyTelling.tsx: Combine GSAP and Framer Motion to create engaging scroll-linked timelines.
  • VirtualCoachTerminal.tsx: Simulates the interactive AI chat built for the marketing hook directly on the homepage.

4. The Desktop Tactical Workspace (frontend_final/app/workspace/)

The workspace is accessed via /workspace. The primary component here is TacticalDashboard.tsx which uses a 60/40 Split View logic combining data rendering with Gemini AI conversational interaction.

Core Tactical Components:

  • Hopper.tsx: The ingestion portal. It mounts an invisible HTML5 input[type="file"], processes uploaded match footage, and establishes a live WebSocket connection to read the pipeline analytics coming from the FastAPI server.
  • VideoHUD.tsx: The split-screen interface mounting the raw uploaded video (using URL.createObjectURL()) adjacent to the telemetry engine.
  • RadarWidget.tsx: Found in components/radar/. This interprets complex JSON positioning artifacts and renders 2D telemetry dots onto a mathematical pitch.
  • useVideoFrameSync.ts: A powerhouse custom React Hook firing on requestAnimationFrame to perfectly sync raw video playtime with the exact frame indices of the JSON telemetry array, completely side-stepping standard state lag.
