Gaffer's Guide: Frontend Architecture Study Guide

The frontend of Gaffer's Guide is a high-performance, dual-purpose Next.js application that handles both the cinematic marketing website and the actual tactical analysis software workspace (designed to look and feel like a modern native desktop app).

The Tech Stack

  • Core Framework: Next.js 16 (App Router) & React 19
  • Language: TypeScript for strict type-safety across components and API responses.
  • Styling: Tailwind CSS v4. We implemented a custom, strict "Dark Room" design system (bg-pitch, text chalk) combined with dynamic emerald-400 neon accents and custom selection highlights (selection:bg-neon/30).
  • Animation Engine: A powerful trifecta:
    • Framer Motion: For buttery UI transitions and element enter/exit animations.
    • GSAP (GreenSock): For complex scroll-based timelines (scrollytelling).
    • Three.js (@react-three/fiber, @react-three/drei): For the physics and canvas background effects.
  • Icons: Lucide React

The codebase is split into two distinct user experiences:

1. The Cinematic Marketing Website (frontend_final/app/page.tsx)

Located in /frontend_final/components/, this side of the app acts as the public landing page. It uses extreme polish and data-driven aesthetics to sell the software.

Where the animations happen & Key Components:

  • OpeningSequence.tsx & TextReveal.tsx: These handle the initial load. They trigger the staggered, cinematic text reveal animations that give the user an immediate premium impression.
  • Hero.tsx & AboutTeam.tsx: The primary landing interfaces.
  • TransformationStages.tsx & ScrollyTelling.tsx: These components combine GSAP and Framer Motion to create engaging scroll-linked timelines, letting the user "scroll through" the data transformation process.
  • VirtualCoachTerminal.tsx: A brilliant component that simulates the actual interactive AI chat of the software right on the marketing page to hook the user.
  • PrivacyMoat.tsx, FeaturesToggle.tsx, & DownloadCTA.tsx: Feature showcases that dynamically explain the local-processing advantage and push the user to download the application.

2. The Desktop Tactical Workspace (frontend_final/app/workspace/)

Located in /frontend_final/app/workspace/components/, this is the actual software interface. While it currently runs in a Next.js browser shell, the UI is built to be wrapped in Tauri or Electron as a native desktop application.

Where the UI logic lives & Key Components:

  • TacticalDashboard.tsx: The core layout engine. It utilizes a 60/40 Split View. The left 60% holds the heavy data (like the TacticalGrid.tsx and InsightCard.tsx), while the right 40% houses the Gemini-style conversational AI for tactical Q&A.
  • VideoHUD.tsx: The star of the workspace. It renders a synchronized dual-feed: the raw match video feed playing exactly in-sync with a 2D radar telemetry plot generated by the data.
  • EngineSettingsModal.tsx: A custom modal providing a 4-mode configuration system. This allows the user to literally hit toggles to decide if the app should use Cloud or Local hardware, and whether it operates Online or Offline.
  • Desktop Shell Components: Components like Sidebar.tsx and Titlebar.tsx strip away the "website" feel and replace it with a collapsible, native app structure to maximize screen real estate for film study.

Custom React Hooks

Found in /frontend_final/hooks/, we built custom hooks specifically to handle the "feel" of processing data:

  • useStreamingText.ts: Simulates the real-time typewriter generation of AI tactical thoughts.
  • useWebSocketProgress.ts: Handles the loading states and telemetry sync progress bars while the "system" claims to be tracking the footage.
