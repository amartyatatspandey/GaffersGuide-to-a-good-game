# Player Reports Data Inventory

This inventory documents all player metrics, telemetry outputs, and structures generated by the Gaffer's Guide tracking and event pipeline. These metrics can be derived immediately in the frontend to support the individual Player Reports module.

---

## 1. Available Output Files

All job-related output files are written to the backend directory (`backend/output/`) upon pipeline completion:

1. **Tracking Data**: `{job_id}_tracking_data.json`  
   *Raw frame-by-frame 2D tracking coordinates, instantaneous speeds, and homography details.*
2. **Threat Profiles**: `{job_id}_threat_profiles.json`  
   *Normalized threat rankings, dominant threat categories, and AI-generated intelligence verdicts.*
3. **Event Index**: `{job_id}_events.json`  
   *Ontology events triggered by positional, movement, and threat-based rules.*

---

## 2. Telemetry and Data Field Specifications

### A. Tracking Data (`{job_id}_tracking_data.json`)
The root structure contains global metadata and a list of frames.

*   `video_path` (*string*): Absolute path to the source footage.
*   `overlay_video_path` (*string*): Absolute path to the annotated composite video.
*   `telemetry` (*object*):
    *   `total_frames_processed` (*integer*): Length of the video in frames.
    *   `frames_standard_homography` (*integer*): Frames projected using normal homography.
    *   `frames_optical_flow_fallback` (*integer*): Frames using fallback motion vectors.
    *   `player_position_space` (*string*): Coordinates unit, always `"meters"`.
    *   `radar_pixels_per_meter` (*integer*): Scale factor, always `10` (10 px ≈ 1 m).
*   `frames` (*array of objects*):
    *   `frame_idx` (*integer*): Current frame index (0-indexed).
    *   `players` (*array of objects*):
        *   `id` (*integer*): Unique player identification number.
        *   `team_id` (*string | null*): `"team_0"`, `"team_1"`, or `null`.
        *   `x_pitch` (*float*): X pitch coordinate in meters (ranges `0` to `105` representing standard pitch length).
        *   `y_pitch` (*float*): Y pitch coordinate in meters (ranges `0` to `68` representing standard pitch width).
        *   `x_canvas` / `y_canvas` (*float*): Bounding box centroid pixels.
        *   `bbox` (*array of 4 floats*): `[x1, y1, x2, y2]` canvas box boundary.
        *   `speed_kmh` (*float*): Instantaneous speed in km/h.
    *   `ball_xy` (*array of 2 floats | null*): Coordinates of the ball in pitch space.
    *   `possession_team_id` (*string | null*): Possessing team ID.
    *   `homography_confidence` (*float*): Quality score of calibration mapping.
    *   `used_optical_flow_fallback` (*boolean*): Indicates if fallback calibration was used.

### B. Threat Profiles (`{job_id}_threat_profiles.json`)
An array of player profiles sorted by threat score.

*   `player_id` (*integer*): Sourced player ID.
*   `team_id` (*string*): `"team_0"`, `"team_1"`, or `"unknown"`. *(Note: Can be cross-referenced with tracking frames to resolve teams if labeled unknown)*.
*   `threat_score` (*float*): Normalized score from `0.0` to `100.0`.
*   `threat_rank` (*integer*): Threat position relative to teammates (1 = highest threat).
*   `primary_threat_types` (*array of strings*): Top 3 contributing ontology rules (e.g. `["THR_001", "THR_006"]`).
*   `explanation` (*string*): Multi-line AI-generated verdict describing their tactical profile.
*   `event_counts` (*object*): Dict mapping event codes to counts.

### C. Event Index (`{job_id}_events.json`)
The timeline of ontology triggers.

*   `events` (*array of objects*):
    *   `event_id` (*string*): Unique UUID for the event.
    *   `event_type` (*string*): Code (e.g. `POS_001`, `THR_002`, `MVT_001`).
    *   `event_name` (*string*): Readable label (e.g. "Dangerous Run").
    *   `category` (*string*): `"positional"`, `"movement"`, or `"threat"`.
    *   `player_id` (*integer*): Player associated with the event.
    *   `team_id` (*string*): Team associated.
    *   `start_frame` / `end_frame` (*integer*): Frame bounds.
    *   `start_time_s` / `end_time_s` (*float*): Time bounds in seconds.
    *   `duration_s` (*float*): Active duration.
    *   `description` (*string*): Narrative summary of the event.

## 3. Derivable Metrics & Calculation Logic

The following player performance metrics can be computed immediately in the frontend from the tracking data payload:

| Metric | Source Fields | Calculation Method |
| :--- | :--- | :--- |
| **Distance Covered** | `x_pitch`, `y_pitch`, `frame_idx` | Sum Euclidean distances $\sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$ between consecutive frames where player is seen (ignore gaps where $\Delta \text{frames} > 50$ or $2\text{s}$). |
| **Average Speed** | Derived Distance, Tracked Frames | $\text{Average Speed (km/h)} = \frac{\text{Distance Covered (m)}}{\text{Tracked Frames} / 25} \times 3.6$ |
| **Maximum Speed** | `speed_kmh` | $\max(\text{speed\_kmh})$ across all frames for that player. |
| **Sprint Count** | `speed_kmh` | Peak-trigger count of contiguous frames where speed exceeds $24.0\text{ km/h}$. Resets when speed falls below $19.0\text{ km/h}$ (hysteresis). |
| **Dominant Threat** | `primary_threat_types` | Sourced from `primary_threat_types[0]`. Mapping provided to human names (e.g., `THR_001` $\rightarrow$ "Dangerous Run"). |
| **Ontology Triggers** | `event_counts` | Aggregate counts for positional, movement, and threat categories. |
